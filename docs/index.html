1. Plan de Pruebas Unitarias

Aseguramiento de calidad y estabilidad del núcleo del sistema

1.1 Objetivo

Asegurar la calidad y el correcto funcionamiento a nivel de código de los componentes individuales de OpenCart. Se validará que las funciones y métodos de los módulos seleccionados retornen los valores esperados ante diferentes escenarios (datos válidos, inválidos y casos extremos).

1.2 Alcance

El plan cubrirá exclusivamente la capa lógica (controladores y modelos) de los siguientes 6 módulos críticos del sistema:

  • Login y Registro: Autenticación, validación de contraseñas, sesiones.
  • Carrito de Compras: Cálculo de subtotales, impuestos, agregar/eliminar items.
  • Checkout / Pago: Procesamiento de datos de facturación y envío.
  • Catálogo y Búsqueda: Filtros de productos, paginación, cálculo de precios con descuento.
  • Gestión de Inventario: Actualización de stock tras una compra.
  • Sistema de Reseñas: Validación de longitud de comentarios y puntuación.

1.3 Herramientas y Entorno

Framework de Pruebas

PHPUnit (Estándar para proyectos en PHP como OpenCart).

Gestión de Tareas

Una prueba unitaria se considerará finalizada cuando cumpla los siguientes requisitos:

El código de la prueba cubre rutas de éxito y rutas de error.

La ejecución en PHPUnit retorna un estado verde (Aprobado).
